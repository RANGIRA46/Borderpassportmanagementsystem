# Glossary

| Term | Definition |
|------|-----------|
| **API** | Application Programming Interface — a set of rules that allow programs to communicate with each other. |
| **BPMS** | Border/Passport Management System — the application described in this repository. |
| **CEPGL** | *Communauté Économique des Pays des Grands Lacs* — an economic community of Great Lakes countries. |
| **CORS** | Cross-Origin Resource Sharing — an HTTP mechanism that allows or restricts cross-origin requests. |
| **CRUD** | Create, Read, Update, Delete — the four basic database operations. |
| **CSP** | Content Security Policy — an HTTP header that helps prevent XSS attacks. |
| **Deno** | A modern JavaScript/TypeScript runtime used by Supabase Edge Functions. |
| **DGIE** | *Direction Générale de l'Immigration et de l'Émigration* — the immigration directorate. |
| **Docker** | A platform for building and running containerised applications. |
| **Edge Function** | A serverless function deployed close to end users on Supabase's global edge network. |
| **GDPR** | General Data Protection Regulation — EU data privacy law. |
| **Hono** | A small, fast web framework for Deno/Node.js/Cloudflare Workers used by the backend. |
| **HSTS** | HTTP Strict Transport Security — forces browsers to use HTTPS. |
| **JWT** | JSON Web Token — a compact, URL-safe means of representing claims between two parties. |
| **KV Store** | Key-Value Store — a simple data store used for application state in the Edge Function. |
| **Laissez-passer** | An emergency travel document issued when a passport is not available. |
| **MRZ** | Machine Readable Zone — the two or three lines at the bottom of a passport data page. |
| **OpenAPI** | A specification for describing REST APIs in a machine-readable YAML/JSON format. |
| **PII** | Personally Identifiable Information — data that can uniquely identify a person. |
| **RLS** | Row-Level Security — a database feature that restricts which rows a user can access. |
| **REST** | Representational State Transfer — an architectural style for designing networked APIs. |
| **SPA** | Single-Page Application — a web app that loads a single HTML page and updates dynamically. |
| **Supabase** | An open-source Firebase alternative providing a hosted Postgres database, Auth, Storage, and Edge Functions. |
| **Swagger** | Tooling for the OpenAPI specification (e.g., Swagger UI for interactive documentation). |
| **TLS** | Transport Layer Security — the cryptographic protocol underlying HTTPS. |
| **UAT** | User Acceptance Testing — testing performed by end users before production release. |
| **Vite** | A fast frontend build tool and development server. |
| **XSS** | Cross-Site Scripting — an injection attack where malicious scripts are inserted into web pages. |
